# Abbreviations

This is a command-only skill.

## Invocation

Use only:

```text
/skill:abbreviations <freeform abbreviation request>
```

Examples:

```text
/skill:abbreviations add 00foo for reading Foo review notes in the task folder
/skill:abbreviations make 00foo mean read the newest Foo notes and summarize blockers first
/skill:abbreviations remove 00foo
```

Do not require exact subcommands. Infer whether the user wants to add, update, clarify, rename, or remove an abbreviation from the freeform request.

Do not auto-use this skill from an ordinary "add abbreviation" request. Wait for the explicit `/skill:abbreviations` command.

## Source of truth

Canonical source:

- `.ai/rules/abbreviations.md`

Runtime/generated consumers:

- `~/.claude/rules/abbreviations.md` is a symlink to the canonical file.
- `~/.pi/agent/AGENTS.md` is a symlink to `.ai/rules/pi-global.md`.
- `.ai/rules/pi-global.md` contains the Pi global rules assembled from the shared rule files; keep its abbreviations section in sync with `.ai/rules/abbreviations.md` until a generator command exists.

Navigator discoverability files, when the abbreviation should be searchable there:

- `.ai/skills-shared/navigator/references/aliases.md`
- `.ai/skills-shared/navigator/references/capability-map.md`
- `.ai/skills-shared/navigator/references/inventory.generated.md` generated by `ruby .ai/skills-shared/navigator/scripts/update_inventory.rb`

## Instructions

1. **Understand the request:**
   - infer the requested operation from the freeform text: add, update, clarify, rename, or remove
   - abbreviation names should normally match `^00[a-z0-9]+$`
   - if the abbreviation name or intended meaning is unclear, ask one concise clarifying question
   - do not ask the user where abbreviations live; this skill owns that routing

2. **Edit the canonical rule file:**
   - update `.ai/rules/abbreviations.md`
   - keep entries in the existing bullet format:
     ```text
     - `00foo` — Short imperative meaning.
       Optional second line with precise behavior.
     ```
   - keep abbreviations sorted near the existing `00...` entries when practical

3. **Sync Pi global rules:**
   - make the matching targeted change in the abbreviations section of `.ai/rules/pi-global.md`
   - do not rewrite unrelated generated sections

4. **Update Navigator only when useful:**
   - if the abbreviation should appear in Navigator's human-facing maps, update:
     - `.ai/skills-shared/navigator/references/aliases.md`
     - `.ai/skills-shared/navigator/references/capability-map.md`
   - run:
     ```bash
     ruby .ai/skills-shared/navigator/scripts/update_inventory.rb
     ```
   - do not invoke `/skill:navigator update` from this skill; make only the small required Navigator edits directly

5. **Verify and summarize:**
   - run `git diff -- <changed paths>` or another targeted diff check
   - summarize changed files and the final abbreviation meaning

## Safety

- Do not edit unrelated skills or rules.
- Do not run specs or RuboCop for abbreviation-only changes.
- If `.ai/rules/pi-global.md` has a real generator in the future, use that generator instead of hand-syncing the section.
